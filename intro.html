Introduction

MControl is a MVC light framework.

MC.Controller class is provided for implementing controller behavior: MVC Controllers are responsible for controlling the flow of the application execution.

Provided features are :

  • interpolated javascript model expression in HTML : {​{ js expression }​}
  • two-way binding beetween input element and model property: m-bind:value="property"
  • event handling binded to controller method: m-on:event=" this.method($event) "
  • collection of dom elements through 'm-ref' modifier labeling labeling"

No other library

MC don't embed third party libraries, feel free to use it in your app code. This choice avoid you conflicts when you embed MC with others open source components and keep MC small.

“If debugging is the process of removing software bugs, then programming must be the process of putting them in.” - Edsger Dijkstra

No Virtual DOM

MC don't use virtual DOM, My opinion is that we don't need (and don't like) to use an other language to explain the HTML I already know. That's why decorating HTML with new attributes ignored by browsers seemed to me less invasive and let your HTML knowledge valid and reusable.

MC Concepts

Controllers are javascript ES6 classes

Modifiers

For MC modifiers are new attributes added to HTML language.

  • All modifiers start with 'm-' prefix to avoid clash name with html standard attributes
    ex: m-control , m-bind, m-class, ...
  • Some modifiers are extended with identifiers to define parameters of the modifier
    ex: mb-bind:value to indicate the node property to binded
  • Some modifiers have a value to describe its behavior
    ex: m-style:text-align="µ.alignement"
